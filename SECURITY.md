# Security Policy

## Supported releases

Petrel 0.2.0 is still a release candidate, so no 0.2.x release is currently
published or supported. When 0.2.0 is tagged, 0.2.x is intended to become the
supported release line; older pre-1.0 releases will not receive security fixes.

## Private reporting

GitHub private vulnerability reporting is not yet enabled for this repository. Enabling it is a pre-tag blocker for Petrel 0.2.0; this document does not claim that the route is currently live.

Once a maintainer enables the repository setting, report suspected vulnerabilities through the repository's GitHub Security Advisory form:

<https://github.com/joshlacal/Petrel/security/advisories/new>

Until that setting is enabled, there is no approved private email address or substitute reporting channel. Do not publish vulnerability details in a GitHub issue or discussion; the release remains blocked until the private route is available.

## What to expect

After the private route is enabled, maintainers will acknowledge reports through the advisory, assess affected releases, coordinate remediation and disclosure with the reporter, and publish supported-version guidance when a fix is available. Petrel does not promise a fixed response or resolution SLA.
